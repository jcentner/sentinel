Recommended Configurations

Privacy-first (all local)

[sentinel]
provider = "ollama"
model = "qwen3.5:4b"
model_capability = "basic"
# Skip test-coherence — 4B model has ~40% FP rate
disabled_detectors = ["test-coherence"]

Best quality (hybrid local + cloud)

[sentinel]
provider = "ollama"
model = "qwen3.5:4b"
model_capability = "basic"

# Route test-coherence to cloud for better precision
[sentinel.detector_providers.test-coherence]
provider = "openai"
model = "gpt-5.4-nano"
api_base = "https://api.openai.com/v1"
api_key_env = "OPENAI_API_KEY"
model_capability = "standard"

No LLM (fastest, deterministic only)

[sentinel]
skip_judge = true
skip_llm = true
# Runs all heuristic + deterministic detectors with no model at all

Recommendations by Situation

Practical advice based on our benchmarks. For untested models, run sentinel benchmark --model <name> to verify quality. The model_capability setting is a hint for prompt strategy — when benchmark data exists, it takes precedence (ADR-016).

Your Situation Recommended Setup
8 GB VRAM, privacy-required qwen3.5:4b — works well for everything except test-coherence. Skip test-coherence or accept noise.
12–16 GB VRAM Larger local model (14B class). Run sentinel benchmark to verify quality — likely better than 4B/9B.
24+ GB VRAM 30B–70B local model. Should perform well — benchmark to confirm and unlock enhanced prompts.
Cloud OK, budget-sensitive gpt-5.4-nano for LLM detectors, local qwen3.5:4b for deterministic. Best cost/quality ratio tested.
Cloud OK, quality-first gpt-5.4-mini or Claude Haiku 4.5 for all LLM detectors.
CPU-only (no GPU) qwen3.5:4b on CPU — slow but functional. Consider cloud API for LLM detectors.
